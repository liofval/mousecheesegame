ネズミがチーズを見つける
強化学習の原理

このドキュメントでは、「ネズミ vs チーズ」ゲームに実装されている強化学習アルゴリズムの すべてを、数学的背景から実装の詳細まで段階的に解説します。

01全体概要

このプロジェクトは、方策勾配法（Policy Gradient）と呼ばれる強化学習アルゴリズムを使い、 15x15のグリッドワールド迷路の中で、ネズミ（エージェント）がチーズ（報酬）を見つけることを学習するシミュレーションです。

KEY IDEA
ネズミは最初、どちらに進めばいいか全く分かりません。ランダムに動き回り、偶然チーズを見つけた経験から 「この状況ではこの方向に動くと良い結果になりやすい」というパターンをニューラルネットワークで学習していきます。

システムの構成要素

02強化学習とは

強化学習（Reinforcement Learning）は、エージェントが環境と相互作用しながら、 得られる報酬を最大化する行動方針（方策）を学習するフレームワークです。

🔁学習サイクル

毎ステップで以下のサイクルが繰り返されます:

STATE
エージェントが環境の現在の状態 s を観測する
ACTION
方策 π(a|s) に従って行動 a を選択する
REWARD
環境が報酬 r と次の状態 s' を返す
LEARN
得られた経験に基づいて方策を改善する
VALUE-BASED VS POLICY-BASED
強化学習には大きく分けて価値ベース（Q学習など: 各状態-行動の価値を推定）と 方策ベース（方策勾配法: 行動確率を直接学習）の2つのアプローチがあります。 このプロジェクトでは方策ベースを採用しています。

03方策勾配法 (Policy Gradient)

方策勾配法は、ニューラルネットワークが直接「各状態でどの行動をどのくらいの確率で選ぶか」を出力し、 良い結果につながった行動の確率を上げ、悪い結果につながった行動の確率を下げることで学習する手法です。

📐REINFORCE アルゴリズム

本プロジェクトで使用している手法はREINFORCEアルゴリズムの変種です。核心となる更新式は:

∇J(θ) = E[ ∇ log πθ(a|s) ・ Gt ]
記号意味このプロジェクトでは
θ ネットワークのパラメータ（重み） W1 (225x200) と W2 (200x4) の重み行列
πθ(a|s) 方策: 状態sでの行動aの確率 Softmax出力の4方向の確率分布
Gt 割引累積報酬（リターン） γ=0.98で割引された将来報酬の合計
∇ log π 方策の対数確率の勾配 y - aprob (選んだ行動の1-hot - 現在の確率)
直感的な理解
良い結果（Gtが大きい）につながった行動の確率を上げ、 悪い結果（Gtが小さい/負の）につながった行動の確率を下げる。 この更新を何千回と繰り返すことで、ネズミは迷路の中で最適な経路を学習します。

04環境: グリッドワールド

ネズミが活動する環境は 15x15 = 225マス のグリッドワールドです。 壁（通行不可）と通路（移動可能）で構成された迷路構造になっています。

🗺マップ構造

🧀チーズの巡回

チーズは4つの固定位置を順番に巡回します。ネズミがチーズに到達すると、 チーズは次の位置に移動します。

順番座標位置
1[2, 2]左上エリア
2[2, 12]左下エリア
3[12, 2]右上エリア
4[12, 12]右下エリア

🎮行動空間

行動ID方向座標変化
0↓ 下[0, +1]
1↑ 上[0, -1]
2→ 右[+1, 0]
3← 左[-1, 0]
壁との衝突
ネズミが壁に向かって移動しようとした場合、位置は変わりません。 これはペナルティにはなりませんが、距離ベースの報酬シェーピングにより 実質的に-1の報酬を受け取ります（チーズに近づけなかったため）。

⏱エピソード

1エピソード = 1000ステップです。エピソード終了時に環境がリセットされ、 ネズミは初期位置 [1, 1] に戻ります。 1000ステップの中で何個のチーズを獲得できたかがエージェントの性能を示します。

05状態表現

ネットワークへの入力は、225次元のバイナリベクトルです。 15x15のグリッドを1次元に展開し、2つの位置に「1」を立てた2-hotエンコーディングを使用します。 

// 225次元のゼロベクトルを作成
const x = new Float64Array(225);

// ネズミの位置をエンコード
x[agentPos[1] * 15 + agentPos[0]] = 1;

// チーズの位置をエンコード
x[cheesePos[1] * 15 + cheesePos[0]] = 1;
なぜこの表現？
各マスが独立した特徴量として扱われるため、ネットワークは位置の関係性を重みから学習する必要があります。 座標値をそのまま入力する方法もありますが、グリッドの各マスに意味がある離散環境では、 one-hot系のエンコーディングが直感的で学習が安定します。

06ニューラルネットワーク

方策ネットワークは2層の全結合ニューラルネットワークです。 状態ベクトルを入力として受け取り、4つの行動それぞれの確率を出力します。

🎲重みの初期化

Xavier初期化（変種）を使用しています。各重みを入力ユニット数の平方根で割ることで、 信号の分散が層を通じて一定に保たれます。

W1[i][j] ~ N(0, 1) / √225   ,   W2[i][j] ~ N(0, 1) / √200

合計パラメータ数: 225 x 200 + 200 x 4 = 45,800 個の学習可能な重み。 隠れ層サイズを変更すると、このパラメータ数も変化します。

07順伝播 (Forward Pass)

入力状態ベクトルからの行動確率の計算は、3つのステップで行われます。

Step 1: 隠れ層の計算

h = x @ W1    →    (1 x 225) @ (225 x 200) = (1 x 200)

Step 2: ReLU活性化

h = max(0, h)
ReLU (Rectified Linear Unit)
負の値を0に、正の値はそのまま通す非線形関数。計算が単純で勾配消失が起きにくいため、 ニューラルネットワークで最も広く使われている活性化関数です。 この非線形性がなければ、2層のネットワークは1層と同等になってしまいます。

Step 3: 出力層 + Softmax

logits = h @ W2    →    π(a|s) = softmax(logits) 
// Softmax: logitsを確率分布に変換
function softmax(logits) {
  const max = Math.max(...logits);   // オーバーフロー防止
  const exp = logits.map(x => Math.exp(x - max));
  const sum = exp.reduce((a, b) => a + b);
  return exp.map(x => x / sum);       // 合計が1の確率分布
}

出力例: [0.15, 0.40, 0.35, 0.10] = 下15%, 上40%, 右35%, 左10%。 ネズミは上に動く確率が最も高く、次に右に動きやすい状態です。

08行動選択: ε-greedy

行動はネットワークの出力確率に従って選択されますが、 一定確率 ε (epsilon) でランダムな行動を取ります。 

if (Math.random() < epsilon) {
  // 探索: ε の確率でランダム行動
  action = randomChoice([0, 1, 2, 3]);
} else {
  // 活用: ネットワークの確率分布からサンプリング
  action = weightedSample(actionProbabilities);
}

⚖探索と活用のジレンマ

EXPLORATION VS EXPLOITATION
活用（Exploitation）: 現在の知識に基づいて最善と思われる行動を選ぶ。
探索（Exploration）: まだ試していない行動を取り、新しい情報を得る。

εが高い（例: 0.2）と、多くの経路を試すため局所解を脱出しやすいが、学習したことを活かせない時間が増える。 εが低い（例: 0.02）と、効率的に動けるが、もっと良い経路を見逃す可能性がある。

📝方策勾配の記録

各ステップで、選択した行動の「ログ確率の勾配」を記録します:

dlog π = y - π(a|s)    where    y = [0, 0, 1, 0] （選んだ行動が「右」の場合）

例えば確率 [0.15, 0.40, 0.35, 0.10] で「右（index=2）」を選んだ場合: 

dlogp = [0-0.15, 0-0.40, 1-0.35, 0-0.10]
     = [-0.15,  -0.40,  +0.65,  -0.10]

これは「右を選んだので右の確率を上げ、他を下げる方向」を示します。 この勾配が後で報酬の大きさで重み付けされることで、良い行動が強化されます。

09報酬設計 (Reward Shaping)

報酬の設計は強化学習の性能を大きく左右します。このプロジェクトでは 2種類の報酬を組み合わせています。

🧀到達報酬（スパース報酬）

イベント報酬
チーズに到達+1.0
それ以外 0.0
スパース報酬の問題
チーズに到達したときだけ報酬が出る場合、ランダムに動くネズミがチーズに辿り着く確率は非常に低く、 学習の初期段階で有用な勾配をほとんど得られません。これがスパース報酬問題です。

📏距離ベースの報酬シェーピング

スパース報酬問題を緩和するため、マンハッタン距離に基づく追加報酬を設計しています:

// 前ステップと現ステップのチーズまでの距離
prevDist = |cheese.x - prev.x| + |cheese.y - prev.y|
currDist = |cheese.x - curr.x| + |cheese.y - curr.y|

if (prevDist > currDist)
  reward += +1    // チーズに近づいた
else
  reward += -1    // チーズから遠ざかった（or 動けなかった）

📊報酬の合計パターン

行動の結果環境報酬距離報酬合計
チーズに到達+1+1+2
チーズに近づいた0+1+1
チーズから遠ざかった0-1-1
壁にぶつかった0-1-1
報酬シェーピングの効果
距離ベースの報酬により、ネズミは初期段階からすぐに「チーズに向かう」という基本的な行動を学習できます。 ただし、壁を迂回する必要がある場合、一時的に遠ざかる行動が最適な場合もあるため、 報酬シェーピングが必ずしも最適解に導くとは限りません。

10割引報酬 (Discounted Rewards)

エピソード終了後、各ステップの「将来を含めた累積報酬」を計算します。 未来の報酬は割引率 γ (gamma) で指数的に減衰させます。

Gt = rt + γ rt+1 + γ2 rt+2 + γ3 rt+3 + ... 
// 末尾から逆順に累積（効率的な計算）
let running = 0;
for (let t = rewards.length - 1; t >= 0; t--) {
  running = running * gamma + rewards[t];
  discounted[t] = running;
}

🔍γ の影響

γ の値100ステップ先の重み性質
0.900.90100 = 0.00003近視眼的。目の前の報酬を重視
0.950.95100 = 0.006やや短期的
0.980.98100 = 0.133バランス型（デフォルト）
0.990.99100 = 0.366長期的。遠い未来も考慮
0.9990.999100 = 0.905超長期的。分散が大きくなりがち

📐標準化（Advantage推定）

割引報酬を正規化して分散を安定させます:

At = (Gt - mean(G)) / std(G)
なぜ標準化するのか
平均を引くことで、「平均より良かった行動」はプラスの勾配、「平均より悪かった行動」はマイナスの勾配になります。 これにより全報酬が正の場合でも適切に良い行動と悪い行動を区別でき、学習が安定します。 これがAdvantage（アドバンテージ）と呼ばれる考え方です。

11逆伝播と勾配計算

エピソード終了後、蓄積した各ステップのデータを使って、ネットワークの重みの勾配を計算します。 これが逆伝播（Backpropagation）のプロセスです。

📉勾配計算の流れ

01
報酬の処理
割引報酬を計算し、標準化してAdvantageを得る
02
勾配のスケーリング
各ステップのdlogpにAdvantageを掛ける
03
W2の勾配
隠れ層の出力と方策勾配から計算
04
W1の勾配
誤差を逆伝播し、ReLUマスクを適用

🔧W2 の勾配

∇W2 += hT @ (dlogp * At)

隠れ層の出力 h (200次元) と、Advantageでスケールされた方策勾配 (4次元) の外積を累積します。

🔧誤差の逆伝播とW1の勾配

δh = (dlogp * At) @ W2T   ⊙   ReLU'(h)
∇W1 += xT @ δh
ReLUの勾配マスク
ReLUの微分は非常に単純です: 順伝播時にh[i] > 0だった要素は勾配をそのまま通し、 h[i] <= 0だった要素は勾配を0にします。これにより、活性化されなかったニューロンは その時点での勾配更新に関与しません。 
// ReLU マスク: 順伝播時にゼロだった要素の勾配をカット
for (let i = 0; i < hiddenSize; i++) {
  if (hidden[i] <= 0) dhidden[i] = 0;
}

📦バッチ学習

勾配は複数のエピソードにわたって蓄積されます（デフォルト: 4エピソード）。 batch_sizeエピソード分の勾配が溜まったところで、まとめてパラメータを更新します。

バッチの意味
1エピソードの経験だけで更新すると、そのエピソード特有のノイズに影響されやすくなります。 複数エピソードの平均を取ることで、勾配推定のノイズが軽減され、学習がより安定します。

12RMSPropオプティマイザ

パラメータの更新にはRMSProp（Root Mean Square Propagation）を使用しています。 勾配の二乗の指数移動平均を追跡し、それを使って学習率を適応的に調整します。

📐更新式

vt = ρ ・ vt-1 + (1 - ρ) ・ gt2
θt+1 = θt + α ・ gt / (√vt + 10-5)
記号意味デフォルト値
ρ (decay_rate)移動平均の減衰率0.97
α (lr)学習率0.0025
gt現在の勾配--
vt勾配の二乗の移動平均0から蓄積
10-5ゼロ除算防止の定数--
なぜ SGD ではなく RMSProp？
通常のSGD（確率的勾配降下法）は全パラメータに同じ学習率を適用しますが、 RMSPropは各パラメータの勾配の大きさに応じて学習率を自動調整します。 勾配が大きいパラメータは小さいステップで、勾配が小さいパラメータは大きいステップで更新されます。 これにより、スパースな入力（2-hotベクトル）の環境でも安定した学習が可能になります。

14プリセット解説

⚡ターボ学習 (FAST)

lr=0.006γ=0.95ε=0.02batch=2hidden=200

設計思想: 高い学習率と小さなバッチサイズで、各エピソードから大きく学ぶ。 低いγにより近くの報酬に集中し、早期に「チーズに向かう」基本動作を習得する。 振動や発散のリスクがあるが、短時間で結果が出るため、素早い実験に向いている。

🏔安定重視 (STABLE)

lr=0.001γ=0.99ε=0.05batch=10hidden=200

設計思想: 低学習率+大バッチで勾配推定のノイズを最小化。 高いγで長期的な報酬パターンを学習し、適度なεで局所解を回避。 収束速度は遅いが報酬曲線が滑らかに上昇する。長期間の学習に適している。

🧠大脳モデル (BEST)

lr=0.003γ=0.995ε=0.03batch=4hidden=400

設計思想: 隠れ層400は標準の2倍の容量。 超高γ=0.995で数百ステップ先の報酬も重視するため、 壁を迂回するような複雑な長期戦略を学習できる可能性がある。 パラメータ数が多いため学習初期は遅いが、最終的な性能は最も高くなり得る。

🗺冒険家 (EXPLORE)

lr=0.003γ=0.98ε=0.20batch=6hidden=250

設計思想: ε=0.20は5ステップに1回ランダム行動ということ。 序盤のスコアは低くなるが、多様な経験を収集するため、 他のプリセットでは見つけられない良い経路パターンを発見する可能性がある。 局所解にハマった時のリスタートにも有効。

🐜最小構成 (TINY)

lr=0.004γ=0.96ε=0.04batch=3hidden=50

設計思想: わずか50ニューロンの隠れ層（パラメータ数: 11,450）。 ネットワークの容量が限られているため、複雑なパターンは学習できないが、 それでもチーズに近づく基本方策は学習可能。「小さなモデルでも学習できるのか？」を実験できる。

🎯師範モード (PRO)

lr=0.003γ=0.99ε=0.03batch=4hidden=300

設計思想: すべてのパラメータを最適なバランスに調整した構成。 適度な学習率、高めのγ、低い探索率、やや大きなネットワーク。 安定性とスコアの両方を追求した、最も実用的なプリセット。 迷ったらこれを選ぶのがおすすめ。

15チューニングのコツ

STEP 1: まず観察する
プリセット「師範モード」で20エピソード以上学習させ、報酬グラフの傾きを観察しましょう。 ヒートマップと方策表示をONにすると、ネズミが何を学んだかが視覚的に分かります。
STEP 2: 問題を特定する
報酬が上がらない → 学習率を上げるか、探索率を上げてみる
報酬が振動する → 学習率を下げるか、バッチサイズを増やす
方策が偏りすぎ → 探索率を上げて多様な経験を収集する
ゆっくりだが確実 → 実行速度を上げて待つか、学習率を少し上げる
STEP 3: 一つずつ変える
複数のパラメータを同時に変えると、どの変更が効果をもたらしたか分からなくなります。 一度に変更するパラメータは1つだけにし、効果を確認してから次の調整に進みましょう。
よくある失敗パターン
